Android Wear

Introdução

Android Wear adota um padrão que favorece pequenas atividades. Seu apelo é fazer o usuário ser mais social, não ficar “preso” ao seu smartphone. Algo que acontece com frequencia é você recorrer ao seu smartphone para verificar uma coisa em específico (um e-mail, por exemplo) e se perder nos milhares de apps e notificações que recebe… acaba até esquecendo o primeiro motivo que o fez pegar o smartphone. Com o Android Wear você não deixaria de conversar numa roda de amigos para verificar um e-mail e acidentalmente acabar dando uma olhadinha no Facebook (se perder lá e deixar de falar com seus amigos que estão bem do seu lado).

Essas micro-interações são criadas com base em padrões conhecidos pelo usuário. Cards, icones, páginas, ações, listas de seleção… Facilita o aprendizado falando a mesma lingua e favorece as interações rápidas.

Android Wear - Notifications

Neste artigo abordaremos uma introdução ao desenvolvimento direcionado ao relógio com Android Wear. Uma das interações com o relógio são as notificações que você recebe no celular e que são exibidas no seu relógio.

Você pode fornecer ao usuário interação com o relógio através de Actions. São botões que você coloca nas Notifications que dão a possibilidade de executar alguma ação sem ter que pegar seu smartphone do bolso. As ações podem ter interatividade por toque ou por voz. Neste artigo vou mostrar a interação com toques nas actions. Algumas imagens para você entender melhor:

Crie seu projetooo:

  • Acesse File> New Project
  • Informe o Application Name, Company Domain, Package Name e Project Location
  • Clique em Next

Na janela seguinte:

  • Selecione Phone and Tablet, pode deixar com Minimum SDK API 14 mesmo.
  • OBS: Neste projeto vamos trabalhar com notificação então por enquanto não precisamos criar o módulo Wear.
  • Clique em Next

Na janela para adicionar activity ao projeto selecione Blank Activity e clique em Next. Na janela de customização da Activity informe o Activity Name, Layout Name, Title e Menu Resource Name. Clique em Finish

Após as configurações você vai ter a seguinte estrutura:

Adicione duas activities para usarmos como destino de nossas ações das notifications. Clique com o botão direito do mouse sobre o package New->Activity->Blank Activity Com isso vamos adicionar uma Activity para controlar a ação de Delete e de Reply para serem o destino dos botões de nossa notificação.

  • MainActivity - tem um botão para acionar a notificação
  • DeleteEventActivity - Abre quando clicarmos em Delete
  • ReplyEventActivity - Abre quando clicarmos em Reply

Na MainActivity colocaremos o seguinte código para exibir a notificação com duas ações:

private void showNotification(){
        // Intent for notification
        Intent mainIntent = new Intent(this, MainActivity.class);
        mainIntent.putExtra("click", "you clicked the notification!");
        PendingIntent mainPendingIntent =
                PendingIntent.getActivity(this, 0, mainIntent, 0);

        // Intent for delete Action
        Intent deleteIntent = new Intent(this, DeleteEventActivity.class);
        PendingIntent deletePendingIntent =
                PendingIntent.getActivity(this, 0, deleteIntent, 0);

        // Intent for reply Action
        Intent replyIntent = new Intent(this, ReplyEventActivity.class);
        PendingIntent replyPendingIntent =
                PendingIntent.getActivity(this, 0, replyIntent, 0);

        // Here you build your notification with the two actions
        NotificationCompat.Builder notificationBuilder =
                new NotificationCompat.Builder(this)
                        .setSmallIcon(R.drawable.android_icon2)
                        .setContentTitle(EVENT_TITLE)
                        .setContentText(EVENT_CONTEXT_TEXT)
                        .setContentIntent(mainPendingIntent)
                        .addAction(R.drawable.delete,
                            getString(R.string.delete), deletePendingIntent)
                        .addAction(R.drawable.reply,
                                getString(R.string.reply), replyPendingIntent);

        // Instance of the NotificationManager service
        NotificationManagerCompat notificationManager =
                NotificationManagerCompat.from(this);

        // Build the notification and pass to manager 
        notificationManager.notify(NOTIFICATION_ID, notificationBuilder.build());
    }

No código acima definimos 3 PendingIntent para tratar as interações com a notificação. É recomendável que se use NotificationManagerCompat, se usar a NotificationManager alguns recursos de WearableExtender podem não funcionar. Podemos abordar WearableExtender nos próximos artigos ;)
